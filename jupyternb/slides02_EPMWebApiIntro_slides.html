Regras de ouro quando se trabalha com Timestamps¶

  • Sempre usar objetos datetime "offset-aware" (delta do local em relao ao UTC).
  • Sempre armazenar datetime em UTC e fazer a converso do fuso horrio apenas ao interagir com usurios.
  • Sempre usar a norma internacional ISO 8601 como entrada/sada para repersentaes de data-hora.
Exemplos norma ISO 8601¶
  • Data: 2019-07-16
  • Data e horas separados, em UTC: 2019-07-16 21:02Z
  • Data e horas combinadas, em UTC: 2019-07-16T21:02Z
  • Data com nmero da semana: 2019-W29-2
  • Data ordinal: 2019-197

  1. Escrita de valores escalares

Para escrever um valor escalar é preciso saber que:¶

  • s possvel escrever em Basic Variables (=Tags) e que NO tenham vnculo com alguma interface de comunicao!
  • tem que informar: valor (condizente com a varivel), timestamp (UTC) e qualidade (OPC UA)
  • o timestamp anterior ao ltimo valor-timestamp-qualiade armazenado vai para o processamento de valores Out of Order - mais custoso!
  • o timestamp coincidente com o timestamp de outro valor-timestamp-qualiade previamente armazenado, promove a substio pelo mais novo!
  • preciso que a sesso de comunicao com o EPM Server tenha sido feita com um usurio com permisses de escrita!

Eu poderia simplesmente "mandar escrever" um valor:¶

  • aleatrio (random);
  • lido de um instrumento de campo via: Modbus, MQTT (paho), AMQP (pika), OPC UA (Free OPC UA), ...;
  • lido de um arquivo: .CSV, .XLSX, .HDF5, ....;
  • lido de um banco de dados SQL(RDB's - PostgreSQL, MySQL, MS SQL Server, ...) | NoSQL (MongoDB, etc.) | ...;
  • proveniente de uma simulao em tempo real (ex.: www.ufrgs.br/reamat/CalculoNumerico/livro-py/pdvi-metodo_de_euler);

da ref. acima: Em Python, podemos computar a soluo numrica deste problema de valor inicial via o mtodo de Euler com o seguite cdigo

#define f(t,u)  
def f(t,u):  
    return 2*u  
#tamanho e num. de passos  
h = 0.2  
N = 6  
#cria vetor t e u  
t = np.empty(N)  
u = np.copy(t)  
#C.I.  
t[0] = 0  
u[0] = 1  
#iteracoes  
for i in np.arange(N-1):  
    t[i+1] = t[i] + h  
    u[i+1] = u[i] + h*f(t[i],u[i])  
#imprime  
for i,tt in enumerate(t):  
    print("%1.1f %1.4f" % (t[i],u[i]))

  1. Escrita de valores histricos

Para escrever "dados históricos" é preciso saber que:¶

  • s possvel escrever em Basic Variables (=Tags) e que NO tenham vnculo com alguma interface de comunicao!

  • preciso que a sesso de comunicao com o EPM Server tenha sido feita com um usurio com permisses de escrita!

ref. gekko

PID Control Tuning A PID Controller has proportional, integral, and derivative terms to determine the controller output ($OP$) based on the set point ($SP$) and process variable ($PV$).

A standard PID form has constants $K_c$, $_I$, and $_D$.

$err = SPPV$

$OP = OP_0 + K_c \, err + \frac{K_c}{\tau_I} \int err \, dt - K_c \, \tau_D \frac{d\,PV}{dt}$
